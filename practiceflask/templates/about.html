About GateDB

Documentation, Setup Guides, and Architecture Overview.

Architecture Vision

GateDB is designed as a standalone tool that serves a dual purpose:

  • Central Governance: Send queries to an Admin-controlled database and track them via an approval queue.
  • Personal Client: Connect seamlessly to your own local databases on your PC and run queries instantly.

Connecting to SQLite

SQLite is file-based. When adding a SQLite configuration in your Settings, follow these rules:

  • Host, Username, and Password: Leave these blank. SQLite does not use them.
  • Database Name (File Path): You must provide the exact absolute path to your `.db` file.

✅ CORRECT Example:

C:\Users\cyber\OneDrive\Desktop\testthis\this_db.db

❌ INCORRECT Example (Do NOT use Python format):

r"C:\Users\cyber\OneDrive\Desktop\testthis\this_db.db"

The 'r' and the quotes are programming syntax, not part of the actual file path. Adding them will cause an error because the computer literally searches for a folder named r"C:.

Connecting to MySQL & PostgreSQL

Unlike SQLite, MySQL and PostgreSQL are Server Engines. To connect to them, you must install them on your PC first.

1. Install the Server

Download and install either MySQL Server or PostgreSQL. During installation, you will set up a master root/postgres user and password.

2. Ensure the Service is Running

The database engine must be actively running as a background service on your PC (usually on port 3306 for MySQL or 5432 for Postgres).

3. Configure in Settings

  • Host: Usually localhost or 127.0.0.1 if it's on your same PC.
  • Username: Your database user (e.g., root for MySQL, postgres for Postgres).
  • Password: The password you set during installation.
  • Database Name: The specific database schema you want to query inside the server (e.g., sakila or my_company_db).
